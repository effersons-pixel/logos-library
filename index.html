Why Google Drive embeds fail — and the fix

Root cause 1 — Google Workspace domain restriction (most common):
If your file is in a work, school, or ministry Google account (not a personal gmail.com), the organisation's admin can block external embedding — even when you set "Anyone with the link." The share settings look correct but embedding is silently blocked at the domain level.

Root cause 2 — X-Frame-Options header:
Google Drive's /preview endpoint sends X-Frame-Options: SAMEORIGIN for many files, which browsers refuse to embed in iframes. This is why all three URL formats (/view, /preview, /uc?export=view) produce the same error.

Root cause 3 — File too large (>25 MB):
Google Drive's inline renderer won't process PDFs above ~25 MB. Large commentaries and study Bibles often hit this limit.

✅ The definitive fix:
Download the PDF → sign in to a personal gmail.com account (not a Workspace account) → upload the PDF there → set sharing to "Anyone with the link" → copy the link → paste it here.

This app's fallback system:
The reader automatically tries three viewers in order: Google Drive → PDF.js (via Google Docs) → Office Online. If the primary viewer fails, it switches automatically. The ↗ Open button always opens the file directly in a new tab as a last resort.

Alternative: use Archive.org or your own host:
Public domain books (Tozer, Sanders, old commentaries) can be uploaded to archive.org for free and embedded with no restrictions whatsoever.
